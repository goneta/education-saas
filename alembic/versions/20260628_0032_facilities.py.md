# 20260628_0032_facilities.py

## Purpose

- Creates `campuses`, `buildings`, `rooms`, `room_equipment` and adds `room_id` to `timetables`, enabling multi-campus scheduling and resource-aware timetabling.

## Local Contracts

- All facility tables are school-scoped. `timetables.room_id` is added as a column + index only (no standalone FK constraint, since SQLite cannot ALTER TABLE ADD CONSTRAINT); the FK is declared on the ORM `Timetable.room_id`/`Room` relationship.

## Verification

- `python -m alembic heads`
- `python -m py_compile backend\models.py`
